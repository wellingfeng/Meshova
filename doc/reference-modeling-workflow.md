# 参考图到程序化模型流程

## 问题归因

- 失败点不是“某个车型没写好”，而是流程把参考图压缩成语义标签，再直接堆 primitive。标签能说明“是什么”，不能约束“在哪里、占多大、和谁相接”。
- 局部结构缺少锚点：开口、负空间、边界线、支撑线、最高/最低/最前/最后点、重复部件数量没有进入代码参数。
- 缺少装配约束：模型把可见区域分类成独立 primitive，但没有说明每个部件连接到谁、嵌入谁、支撑谁，导致玻璃、面板、尾部、附属结构出现空气缝或悬空。
- 验收过粗：只看整体像不像，缺少多视图、局部部件、位置比例的对照。
- 用户标注图里的箭头/圈选没有转成约束，导致被指出的问题只在单个模型里手补，没进入生成闭环。

## 通用流程

1. 先抽锚点，不先写模型：主轮廓、支撑/接触/底线、局部轴向极值、主要截面宽高、开口/负空间、边界线、重复数量、对称关系、独立构件。
2. 抽装配图，不先堆零件：每个可见部件必须声明 attachment，说明它接触、嵌入、贯穿、支撑或悬挂在哪个主结构上；没有参考依据时默认相邻部件应连接或轻微嵌入。
3. 再分层建模：连续主壳/骨架 -> 开口/负空间 -> 嵌入式面板/附属结构 -> 表面细节 -> 材质。禁止先堆语义部件名。
4. 每个关键锚点映射到参数或截面点。无法参数化的锚点至少落到 named part，便于后续反馈定位。
5. 连接优先策略：同一连续表面的颜色/材质分区用共面或微内嵌面表示；真正分离的部件才留空隙；开口、窗、壳体、支架、尾端、前端模块必须贴合主结构。
6. 渲染验收使用参考图最相关的视角：正交主视图、侧视图、俯视图、透视图按物体选择；有单张照片时补一个 photo-matched 透视。
7. 反馈轮先修最大几何误差：位置、比例、数量、层级、连接关系。小细节和材质后置。
8. 类别规则只作为临时模板：车辆、家具、角色、建筑可以各有 checklist，但不能混进全局 prompt，除非规则能泛化到所有物体。

## 分级评估与候选门禁

- D0 归一化轮廓：只判断形状，忽略平移与缩放，快速暴露大比例错误。
- D1 原画布形状：联合 canvas IoU、edge F1、bbox IoU、中心偏差，防止错误相机和错误构图靠自动对齐得高分。
- D2 外观：颜色仅作次级信号，不得覆盖形状回退。
- D3 正式候选：融合形状、外观与多视角 solidity；达到阈值才允许早停。
- 第一份有效结果作为 baseline。后续候选必须提高目标分，同时已锁定的归一化轮廓、原画布轮廓、边缘和 bbox 不得超过容差回退。
- 被拒候选仍进入反馈，但不能覆盖 accepted best。最终输出只取 accepted best，不取最后一次生成结果。

## 通用验收清单

- 轮廓：整体 silhouette 是否匹配，长宽高比例是否偏。
- 极值：最高/最低/最左/最右/最前/最后点是否落在正确部件上。
- 支撑：接触面、底线、悬空关系是否对。
- 装配：主要部件是否形成合理连通图；相邻部件是否贴合、嵌入或共享边界；是否存在无依据悬空。
- 开口/负空间：洞、窗、缝、透明面、镂空区域是否分片且位置正确。
- 部件层级：主结构、附属结构、表面细节没有混层。
- 数量/节奏：重复元素数量、间距、对称/非对称关系是否对。
- 边界：颜色线、材质分界、接缝只在影响形体时变成几何；否则做浅层表面细节。
- 坐标：模型居中、尺度稳定、朝向和 viewer 预设视角一致。

## 动物/马类附加规则

- 先建类别模板：骨架点、躯干截面、颈弧、头骨截面、四肢关节、蹄/爪、鬃毛、尾巴；不要从零自由堆球。
- 四足动物走专用 `quadruped` 模板：一条 body/neck/head 中轴 + 多截面 skin；四肢是骨架折线 + 锥形截面；脚端独立蹄/爪。
- 主体连续：躯干、颈、头优先合成一个连续 `body_skin`；肩胛和臀部靠截面变化表达，避免两个大球。
- 四肢必须有结构：前/后腿、左右腿、脚端都要命名，脚端必须接近地面；失败时 scorer 能指出缺哪类部件。
- 材质分离：短毛动物躯干用 `shortCoat` 或 `blackCoat`；`fur` 只给毛绒玩具/长绒近似；鬃毛和尾巴用 `hair`。
- 自动 gate：类别模板导出前跑 render-free scorer；必须检查连续体表、侧投影、四足布局、脚端接地、材质。截图只做视觉复核，不能作为唯一通过条件。

## 四足动物参考来源

- Teddy / SIGGRAPH 1999：用自由轮廓 stroke 生成 3D freeform 形体，说明动物 blockout 应先抓侧影再膨胀截面。
- Morphable model of quadrupeds skeletons：四足动物可用少量直观测量控制 3D 骨架，Meshova 对应为 preset + 骨架/截面参数。
- Pinocchio / SIGGRAPH 2007：动画角色应先有内部骨架，再把表皮附着到骨架；Meshova 当前阶段用确定性截面 skin 近似，不做完整 skin weight。
