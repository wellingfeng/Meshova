Web 程序化建模 + 程序化材质 ｜ 脚本化 DSL · WebGPU · headless 截图 · AI 自迭代闭环 ｜ 开源 / 自研重写避 GPL

项目背景 核心理念 总体架构 分阶段计划 DSL 设计 几何函数 材质 / PBR 截图闭环 里程碑 技术选型 关键决策
一句话定调：这是一个项目两条产出线。几何与材质共用同一套内核（脚本化 DSL + WebGPU + 截图闭环 + AI 编排）。 几何先打通验证闭环（P0–P4），材质作为低成本增量挂上去（P5–P7）。先让 AI 自迭代回路在几何上跑活，材质几乎顺水推舟。 真正的壁垒不是几何算法数量，而是「让 AI 一写就对的 TS DSL」和「让 AI 看着截图改脚本的自迭代回路」。

核心理念

为什么脚本化而非节点连线

  • 可读脚本是 AI 的母语，训练语料 99% 是代码；节点图序列化后又长又绕，AI 要在图结构和语义间反复翻译。
  • 可写控制流（循环/条件/递归/函数）用语言原生语法；节点图表达这些极笨重。
  • 可测diff 友好，改一行就是一行，AI 自我迭代定位精准。

三大原则

  • AI 生成「配方」而非「产物」：AI 写脚本调用几何/材质函数，不直出网格或贴图。结果可参数化复改，不是黑盒。
  • 确定性：seed 化一切随机。同脚本同 seed → 同结果。截图测试与 AI 复现全依赖它。
  • 视觉反馈闭环：AI 写脚本 → 执行 → 截图 → 多模态自评 → 改。这是当前 text-to-3D 做不到的核心创新。

DSL 设计（成败关键）

核心工作不是「实现多少算法」，而是「把 API 设计得让 AI 一写就对」。几何与材质同一套哲学：

  • 函数式 / 不可变优先：无隐藏状态，AI 推理无副作用。链式或嵌套二选一，贯彻到底。
  • 命名即文档：scatterOnSurface(mesh, count, seed) 比 scatter(m,n,s) 对 AI 友好十倍。
  • 强类型(TS)：类型签名就是给 AI 的 API 契约，大幅降低幻觉；LSP 给 AI 自动补全级约束。
  • 小而正交的核心集：组合 > 数量。几何 10 个 + 材质 8 个就能跑通全闭环。
  • 脚本 = 受限 TS：调你的库，别造新语法（AI 没语料）。

几何脚本示例

const m = extrude(
  subdivide(box(1, 1, 1), 2),
  0.2
);
const tower = array(m, { count: 8, axis: 'y', step: 1.1 });
return scatterOnSurface(tower, { count: 40, seed: 7 });

材质脚本示例

const rust = blend(
  metalBase({ roughness: 0.3 }),
  rustLayer({ scale: 4, seed: 7 }),
  { mask: fbm({ scale: 8, octaves: 5 }) }
);
// 输出: { baseColor, metallic, roughness, normal, ao, height }
return rust;

生成路径（建议混合）

  • GPU 程序化（主路）：纹理函数映射成 WGSL，fragment 或 compute 逐像素算。快、分辨率无关、契合 WebGPU compute。
  • CPU 兜底：复杂逻辑或调试时 CPU 算小图。先 CPU 跑通正确性，再 GPU 提速。
  • normal/height 自动派生：由 height 场求导生成 normal，减少 AI 负担。
PBR 正确性是 TA 背景的发力点：能量守恒、metallic/roughness 物理范围约束写进类型或运行时校验，挡住 AI 生成的非法值。 预览必须用标准 PBR + IBL，否则截图自评失真，AI 会学歪。

几条要钉死的决策

  • 必守几何先行，材质复用：别并行铺两摊。P0–P4 把内核和闭环磨好，P5 起材质增量很小。
  • 必守确定性是底线：seed 化一切随机，否则截图测试和 AI 复现全崩。
  • 必守沙箱执行：AI 写的脚本必须隔离跑，防死循环/恶意代码。
  • 范式脚本=受限 TS 调库，别造 DSL 新语法。
  • 范式每个函数自带最小自测脚本 + 基准截图：既是测试，也是喂 AI 的 few-shot 范例库。
  • 范式先少后多：10 几何 + 8 材质函数跑通全闭环，胜过堆量。
  • 合规开源用 MIT/Apache，自研重写避 GPL，反而扩大影响力。
