结论先行

  • Houdini 的强点不是单个算法，而是属性流 + 命名选择 + 可视阶段边界。Meshova 应把 group/name/uv/material/seed/report 作为一等数据，不只传 positions/indices。
  • 程序化建模常见骨架：输入参数 -> 粗形/曲线骨架 -> 分组/属性标注 -> 局部规则 -> 复制/循环 -> 清理验证 -> 输出缓存。
  • AI 闭环最该学 Houdini 的是可观察性：每个阶段有可命名输出、可缓存、可截图、可测 bbox/面积/连通块/法线/非流形。
  • 曲线流程最适合 Meshova 下一阶段补齐：curve/resample/polyframe/sweep 能覆盖头发、绳索、管道、植物、轮廓装饰、家具边线。
  • 不要照搬节点 UI。照搬网络语义：稳定输入输出、属性表、局部表达式、显式随机种子、阶段验证。

Curve -> Resample -> Frame -> Sweep

曲线先确定骨架，再重采样稳定密度，再求切线/法线 frame，最后扫掠截面。适合头发、管线、枝条、装饰线、轮廓条。

Group/Name 贯穿选择语义

复杂模型不靠索引猜部位，而靠 group/name 属性流转。Meshova 选择器应输出命名面集/点集，后续 extrude/material/validate 共用。

FileCache/ROP 固化昂贵阶段

昂贵节点有缓存边界。Meshova AI 闭环也要缓存 hash(params+seed+version) 的中间 mesh/material，截图迭代才快。

filecacherop_geometry

Meshova 建议路线

短期：属性与选择系统

新增 face/point/group/name/material 属性表。所有几何算子保留属性映射。选择器返回命名集合，不返回临时索引。

短期：曲线建模栈

实现 polyline/Bezier/NURBS-lite、resample、frame transport、sweep profile、curve loft。优先稳定、可测、可截图。

短期：质量报告

每次 build 输出 bbox、surface area、volume 近似、连通块、退化面、非流形边、normal flip、UV 缺失率。

中期：实例系统

scatter 返回点属性；copyToPoints 输出 instance buffer。AI 调参时不必每次烘大 mesh，截图前再 bake 或 GPU instance。

中期：阶段缓存

算子链节点 hash = op + params + seed + input hash。缓存中间 mesh/material，失败时定位到具体 stage。

中期：SDF/体素接口

布尔、融合、厚度、形体约束可先走 SDF。先提供抽象接口和验证，不急于复杂三角布尔完全体。

推荐 DSL 形状

Houdini 是节点图；Meshova 应保持脚本优先。但 DSL 可吸收节点图的 stage、group、attribute、cache、validation。

const body = stage("body.blockout", () => box({ size }).bevel(radius).tagFaces("front", byNormal(zPlus)) ); const ribs = stage("detail.ribs", () => curveSpine(points) .resample({ segments: 48 }) .frame({ mode: "parallelTransport" }) .sweep(profile.roundRect({ width, height })) .copyAround(body.group("front"), { count, seed }) ); return validate(merge(body, ribs), { maxDegenerateFaces: 0, requireGroups: ["front", "ribs"], screenshotViews: ["front", "side", "iso"], });
