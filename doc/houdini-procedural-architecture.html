1为什么用程序化做建筑

建筑是程序化建模最甜的场景：高度模块化、规则性强、需要批量与可调。

建筑天然适合程序化

  • 规则性强：楼层等距重复、立面网格化、门窗按开间排布，全是可参数化的规律。
  • 模块化：墙体、窗、阳台、屋檐、斗拱都是可复用模块，copy to points 一铺就成。
  • 需要批量：一条街、一座城上千栋楼，手 K 不现实，规则生成才划算。
  • 反复改：甲方改层高、改风格，改一个参数全场景重算，这是手工建模给不了的。

核心思想：不是建模，是写"生成规则"

程序化的产物不是一坨网格，而是一套从输入到模型的规则链。改输入（轮廓线、层数、随机种子）→ 重跑 → 出新模型。

一句话
手工建模交付"鱼"，程序化交付"渔"——一个能反复生成鱼的生成器（HDA / 脚本）。

这与 Meshova "输出永远是可重跑脚本，不是烘焙网格" 的核心理念完全一致。

两条主流落地路线
① 纯 Houdini 内做完整建筑（影视、渲染、破坏，如《湖边小屋》《科幻城市》）。 ② Houdini → UE5 (PCG/HDA)：Houdini 生成逻辑封装成 HDA 或喂给 UE5 PCG，美术在引擎里实时刷建筑（游戏大世界主流，如沐月、欧酱、wangtaian 的城市系列）。两条路前半段建模思路通用。

2建筑程序化主流程

从一条轮廓线到成品建筑，几乎所有教程都遵循这条"自底向上分层"的管线。
01 INPUT
基础轮廓
建筑占地的闭合曲线 / 矩形底面。可手画、可来自 GIS/OSM 矢量。
curve primitive OSM
02 MASS
体块 / 楼层
轮廓沿 Y 拉伸成体块，按层高切分出 N 个楼层。
polyextrude resample for-loop
03 FACADE
立面分格
每层墙面按开间细分成"单元格"，每格是一个可放模块的槽位。
divide copy to points
04 MODULE
模块装配
按规则往槽位放窗/门/墙/阳台模块，转角与边缘特殊处理。
copytopoints switch attr
05 ROOF
屋顶
顶面轮廓生成坡屋顶/平顶/中式檐口，古建在此做曲面与瓦。
PolyExpand2D sweep
06 DETAIL
细节 / 做旧
挑檐、栏杆、管线、噪声做旧、随机变体打破规整感。
noise scatter rand
07 OUT
输出 / HDA
封装参数成 HDA / 导出到引擎，UV、LOD、碰撞一并处理。
HDA UV PDG
关键认知：自底向上 + 分层属性驱动
每一层都靠属性（attribute）传递信息：点上记 floor_id、is_corner、module_type、orient，下游节点只读属性决定放什么、怎么转。属性设计好，整条链就清晰；属性混乱，后面全是 hack。

3建筑必备核心节点

不求全，只列建筑流程里出镜率最高、必须吃透的那批。
节点 / 概念作用建筑里的典型用法
curve / resample画轮廓、把曲线均匀打点建筑底面轮廓；resample 后每个点 = 一个开间槽位
polyextrude面沿法线挤出轮廓 → 体块；墙面 → 厚度；窗洞内凹
copy to points把模块复制到每个点上立面铺窗、楼层堆叠、栏杆/瓦片阵列（建筑装配的心脏）
for-each loop对每个楼层/每面墙循环处理逐层切分、逐面分格、逐栋应用规则
attribute wrangle (VEX)写代码批量算属性判转角、定模块类型、控旋转 orient、随机变体
group / blast选面/删面选顶面做屋顶、选侧面做立面、删内部面
PolyExpand2D2D 轮廓内缩/外扩墙体厚度、屋檐挑出、道路/地块偏移
switch / rand多输入随机选择窗户样式随机、楼栋风格切换、打破重复感
noise / fbm程序化噪声做旧、墙面起伏、楼栋高度随机分布
sweep截面沿曲线扫掠屋脊、檐口线脚、栏杆扶手、管线
HDA (Digital Asset)把网络封装成带参数的资产暴露层数/风格/种子等参数，给 UE5 或他人复用
VEX 是分水岭
纯连节点能做 70% 的建筑，但转角判断、模块类型分配、精确旋转这些规则逻辑离开 VEX 就会很笨拙。《湖边小屋》《玩转 VEX》系列反复强调：程序化建模真正的"心法"是用属性思考 + 用 VEX 写规则，节点只是容器。矩阵 / 四元数 orient 控制朝向是建筑装配的高频考点。

4各阶段细节要点

每一步真正容易出问题、教程反复提醒的地方。

① 轮廓 & 体块

  • 轮廓曲线缠绕方向要统一（CCW），否则法线朝里、挤出方向反、UV 翻面。
  • resample 的分段长度即开间模数。先定模数（如 3m 一开间），再反推分段，立面才对齐。
  • 体块阶段先用包围盒/简模占位确认比例和层数，别一上来抠细节。

② 楼层切分

  • 层高用参数，不要写死。首层（商业/大堂）常比标准层高，单独处理。
  • 每层打上 floor_id 属性，顶层/底层/标准层走不同分支（顶层接屋顶，底层接入口）。

③ 立面分格（最关键）

  • 把每面墙变成规则单元格网格，格心放点，点上记朝向 N / orient。
  • 转角处理是难点：角点要么放专门的转角模块，要么让相邻两面避让，否则模块穿插。用 VEX 判 is_corner。
  • 窗墙比、门的位置（一般底层中间）用属性控制，不要硬编码索引。

④ 模块装配

  • copy to points + orient 四元数让模块对齐墙面法线，这是朝向正确的核心。
  • 模块自身建在统一的局部坐标原点（如以底边中心为原点），否则复制后到处错位。
  • 用 switch+rand(种子) 做样式变体，种子固定 = 结果可复现。

⑤ 屋顶（中式/古建尤其重）

  • 平顶最简单（顶面直接封）。坡屋顶用 PolyExpand2D 做屋脊骨架再起坡。
  • 中式檐口要曲面起翘，毛球道人方案提到：Houdini 里先做平屋顶逻辑，曲面/起翘放到 UE 二次加工（GPU），分工更稳。
  • 瓦片是典型 copy to points 沿屋面阵列；斗拱、瓦当做成可换模块。

⑥ 细节 & 做旧

  • 规整是程序化的"塑料感"来源。用 noise 给位置/旋转/颜色加微扰动打破完美。
  • scatter 撒管线、空调外机、杂物、植被，城市感全靠这层。
  • 变体一定走种子，不要用 Math.random 式真随机，否则每次重算都变。

⑦ 输出 / 落地

  • UV：程序化模型的 UV 要程序化生成（按模块/按面），别指望手摆。
  • 到引擎要管 LOD、碰撞、合批；UE5 用 PCG/HDA 时注意 WorldPartition 分块。
  • 大城市用 PDG/TOPs 批处理多栋楼并行烘焙（魔猿、wangtaian 城市流程）。

5工程规范（做对的习惯）

教程里 UP 主反复踩坑后总结的"该这么做"。

该做的

  • 参数化一切：层数、层高、开间、风格、种子全部暴露成参数，不写死。
  • 用属性驱动：信息记在点/面属性上，下游只读属性决策，逻辑清晰可追。
  • 种子可控：所有随机走显式种子，保证同输入同输出（可复现）。
  • 模块归一化：每个模块统一局部原点与朝向约定。
  • 分层分网络：体块/立面/屋顶/细节各自子网络，便于调试与复用。
  • 命名规范：节点和属性起可读名字（floor_id 而非 attr1）。
  • 先粗后细：先占位简模确认比例，再逐层加细节。
  • 封 HDA：成熟逻辑封成资产，参数清晰、可给引擎/他人用。

该避免的

  • 硬编码索引：靠 point 序号 hack，换输入立刻崩。
  • 真随机：用不带种子的随机，结果不可复现。
  • 巨型 wrangle：一个 VEX 节点塞几百行，无法调试。
  • 过早抠细节：比例没定就堆窗台线脚，改一次全废。
  • 忽略转角/边界：只测中间整齐区，转角和收边一看就穿帮。
  • 不管法线缠绕：UV、挤出、着色全错的根源。
  • 烘死网格：交付一坨 mesh 而非生成器，失去程序化全部价值。
命名/属性约定示例
楼层 i@floor_id、转角标记 i@is_corner、模块类型 s@module（window/door/wall/balcony）、朝向 p@orient（四元数）、变体种子 i@seed。一套约定贯穿全网络，新人也读得懂。

7对 Meshova 的启发

把 Houdini 建筑方法论映射到 Meshova 的脚本化 DSL。

理念高度一致

  • Houdini 输出 HDA/网络，Meshova 输出可重跑脚本——都是"交付生成器不交付死网格"。
  • Houdini 靠点/面属性驱动，Meshova 已有 coloredPart 按几何字段着色，思路同源，可扩展成属性驱动装配。
  • Houdini 强调种子可复现，正是 Meshova 的硬不变量（同 seed 同结果）。

可借鉴的 DSL 算子

  • copyToPoints / InstancePlan 已有 → 建筑模块装配的基石，对齐 Houdini 心脏节点。
  • 建议补立面分格算子：把一个面拆成 m×n 单元格、格心出点带朝向。
  • 建议补转角/边界标记字段，配合 selectByAttr 做规则分配。
  • 沿曲线 sweep 已有 → 做檐口/栏杆/线脚。
  • AI 脚本可直接写"分层 → 分格 → 按规则放模块"的建筑生成器，正好是 Meshova 闭环的高价值题材。
落地建议
做一个 building 示例模型：参数 = 楼层数 / 层高 / 开间数 / 窗样式种子。流程严格按本页"分层→分格→装配→屋顶"，用现有 copyToPoints + 字段着色 + 种子 PRNG 实现，正好验证 Meshova 几何核对"规则性建筑"的表达力。
