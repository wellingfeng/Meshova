Maya 可学
Modeling Toolkit 的“选择组件 → 激活工具 → 调整工具参数”流程。对 Meshova 等价于：Selection + OpOptions + 可重放操作历史。
3ds Max 可学
Editable Poly 的 sub-object levels 与 modifier stack。对 Meshova 等价于：拓扑域明确、算子可堆叠、每步可序列化。
Blender 可学
Mesh Edit + Modifier 的参数细度：bevel width/segments/profile/clamp/harden normals，bridge/loop cut/knife 都有明确拓扑语义。
Meshova 方向
不要克隆 DCC UI。抽象成脚本 API：selectEdgesByAngle() → bevelEdges() → solidify() → computeNormals()。

DCC 建模流程可借鉴点

软件典型流程核心工具给 Meshova 的启发
Autodesk Maya 组件模式选择 face/edge/vertex，Modeling Toolkit 中执行 Extrude、Bevel、Bridge、Cut 等工具，工具参数可交互调整。 Extrude、Bevel、Bridge、Multi-Cut/Cut、Merge、Target Weld API 不只要 mesh-in/mesh-out，还要 Selection、工具参数、重复执行、局部坐标、keep faces together。
Autodesk 3ds Max Editable Poly 按 Vertex/Edge/Border/Polygon/Element 操作；Edit Poly / Modifier Stack 让修改以层级方式保留。 Extrude、Bevel、Inset、Outline、Chamfer、Connect、Bridge Meshova 应有 sub-object domain：顶点、边、边界、面、元素。后续可做 ModifierStack 或 OpPlan。
Blender Edit Mode 中做 mesh editing；复杂形体常由 edit ops + modifiers 组合，例如 Bevel、Solidify、Mirror、Subdivision。 Extrude Faces、Inset Faces、Bevel、Bridge Edge Loops、Loop Cut、Knife 函数签名应暴露 width/segments/profile/clamp/normals/material 等选项；不要只有一个 distance。

共同底层模式

组件选择→ 拓扑邻接→ 局部 frame→ 生成新边/面→ 法线/UV/材质继承→ cleanup/诊断

这些工具表面上是倒角、挤出、切线；底层都要求 mesh 能回答：某条边相邻哪些面、某个 face region 的边界是什么、哪些边形成 loop、哪些点应 weld、哪些新面继承哪个 material slot。

P1：硬表面核心算子

函数参考工具关键参数为什么先做
bevelEdges(mesh, selection, opts) Blender Bevel / Maya Bevel / Max Chamfer width, segments, profile, clampOverlap, miter, hardenNormals, material 硬表面质感核心。没有倒角，机械、建筑、家具都廉价。
insetFaces(mesh, faces, opts) Blender Inset Faces / Max Inset amount, individual, evenOffset, boundaryOnly 门窗、面板、按钮、台阶、屋顶边框都需要。

bridgeLoops(mesh, loopA, loopB, opts) Bridge Edge Loops / Maya Bridge / Max Bridge segments, twist, profile, smoothness 连接管道、孔洞、机械外壳、角色肢体段。
solidify(mesh, opts) Blender Solidify / Max Shell thickness, offset, evenThickness, rim 薄片变壳体。衣服、外壳、叶片、金属板必备。

建模流程缺口

1. 当前函数像“直接生成最终 mesh”

AI 写代码时只能改数字。DCC 流程更像“选择 → 操作 → 参数 → 历史”。Meshova 需要中间态：Selection、TopoMesh、OpPlan。

2. 当前选择太弱

selectByAttr() 可选法线/高度，但不够表达“选硬边”“选边界 loop”“选第 3 层窗户外框”。要补 selection DSL。

3. 当前 extrude 不像 DCC

逐三角挤出会产生每个三角独立小棱柱。DCC 的 region extrude 需要保持连通区域、生成共享 side walls、支持 taper/offset。

4. 缺“可修复失败”的诊断

程序化建模会经常遇到自交、重叠、零面积、非流形。算子应返回 warning，而不是静默生成坏 mesh。

建议的数据流

Mesh→ TopoMesh→ Selection / Group→ Operator + Options→ Warnings→ Mesh + debug attrs

这条链适合代码、Viewer UI、AI agent 共用。Viewer 可以显示 selection/group/warning；AI 可以根据 warning 改参数；最终仍输出普通 Mesh。

建议先做的垂直 demo

Demo覆盖算子价值
硬表面工具箱盒insetFaces、extrudeRegion、bevelEdges、computeNormals最小闭环。能直接证明 Meshova 从 primitive 拼接升级到 DCC 建模。
