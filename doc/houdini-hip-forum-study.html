3逐类拆解：能学什么、怎么映射到 Meshova

造型 / Primitives 现在就做

观察到：box / sphere / grid / tube / circle / curve / revolve 是所有工程的地基。真实工程几乎从不直接用最终基元，而是基元 + matchsize 对齐 + 立即接拓扑算子。

映射到 Meshova：Meshova 已有 box/sphere/plane。优先补齐 tube、circle、curve、revolve（车削）——它们是组合复杂造型的最高频起点。matchsize（按包围盒对齐/缩放）值得做成工具函数，AI 拼装多部件时极其有用。

拓扑算子 / Ops 下一梯队

观察到：polyextrude(48)、polysplit、polybevel、blast(删面/点)、smooth、clean、fuse 构成日常工作主体。polyextrude 出现频率仅次于基元，说明挤出是程序化造型的一等公民。

映射到 Meshova：Meshova 已有 extrude/subdivide/displace。下一梯队：bevel（倒角）、blast（按 group 删除）、fuse（按距离焊点）、clean（去重/退化清理）。bevel 对硬表面造型质量提升最大。

复制 / 散布 现在就做

观察到：copytopoints(33) 和 scatter 是程序化的灵魂——在采样点上实例化几何。配合 pack/xformpieces 处理大量副本，foreach 做逐元素循环。

映射到 Meshova：Meshova 已有 copyToPoints/scatter/InstancePlan，方向完全正确。要补的是 per-point 属性驱动实例的变体（用点上的 scale/orient/id 选不同源几何），这正是 copytopoints 真正强大的地方。

曲线 / 重采样 现在就做

观察到：resample(32) 高频出现，说明曲线工作流的关键是先把曲线重采样成均匀点，再做扫描/挤出。carve（截取曲线段）、sweep（沿线扫面）、polywire（线变管）配套使用。

映射到 Meshova：Meshova 已有 curves/sweep。强烈建议补 resample（按段长/段数均匀重采样） 和 polywire（曲线→圆管）——做绳索、树枝、管线、藤蔓的通用利器，且实现简单。

属性 / 字段 现在就做

观察到：attribwrangle(130) 是绝对核心——它就是 VEX 代码节点。再加 attribcreate / attribpromote（点↔面属性传递）/ attribransfer / measure / connectivity（连通分量编号）。整个 Houdini 的可编程性都压在 wrangle 上。

映射到 Meshova：这对应 Meshova 的几何字段(field)系统，是重中之重。AI 脚本要能像 wrangle 一样：读 @P/@N、写自定义属性、按属性着色(coloredPart 已支持)。建议把 connectivity（islands 编号）、measure（面积/曲率）做成字段函数，喂给 scatter/select。

地形 / HeightField 长期/暂不

观察到：heightfield_* 系列（noise/erode/mask/xform）是 Houdini 地形管线，本质是2D 高度场 + 噪声叠加 + 侵蚀模拟，最后转网格。

映射到 Meshova：和 Meshova 的 displaceByNoise + texture/patterns 噪声同源。地形可作为一个高价值 demo：grid + fbm 高度场 + 简单热侵蚀 → 山地。复用现有 noise/fbm，不需新内核。

材质 / Shading 下一梯队

观察到：principledshader + MaterialX(mtlxstandard_surface / mtlximage / mtlxnormalmap / mtlxmix / mtlxdisplacement) 主导。MaterialX 是节点化 PBR，标准面=base/rough/metal/normal/displacement 通道，和 Meshova 的 7 通道 Material 几乎一一对应。

映射到 Meshova：验证了 Meshova 材质内核的通道设计正确。可学 MaterialX 的节点组合方式：mtlxmix(按 mask 混两种材质)、mtlxplace2d(UV 变换)、colorcorrect。这些正好是 texture/filters 该补的组合算子。

解算 / Sim 长期/暂不

观察到：vellumsolver/constraints(布料/软体)、popforce、dopnet。这是动态模拟，本质非确定、依赖时间步。

映射到 Meshova：明确不进 Meshova 范围——和「确定性、同种子同结果」的硬约束冲突。但 Vellum 的约束松弛思想(迭代求解距离约束)可借鉴做静态形变，比如让程序化绳索/布料一次性松弛到稳定形状（确定性迭代，非逐帧）。

Rig / KineFX 长期/暂不

观察到：kinefx::skeleton / rigpose / addjoint / jointcapture、APEX 图。现代 Houdini 把骨骼也做成了程序化节点流。

映射到 Meshova：超出 Meshova 当前 PBR+造型范围。仅记录方向：未来若做可摆姿势的程序化角色，骨骼绑定也能脚本化、确定性生成。短期忽略。

5工作流与验证经验

A. 节点命名与网络组织

真实工程大量出现 null(229)、postitnote(便签)、netbox(分组框)、switch(73,A/B 切换分支)。专业 TA 用 null 节点做"命名锚点/接口"，用便签写意图。启发：Meshova 脚本应鼓励 part 命名规范 + 注释，AI 生成脚本时保留可读结构，便于自迭代时定位修改点。

B. 非破坏 + 可回溯 = 程序化的本质

整个 hip 是一棵"算子依赖图"，改任意上游参数全链重算。这正是 Meshova "脚本而非烘焙网格"理念的工业级印证：输出永远是可重跑的过程，不是死数据。Meshova 的不可变网格约定 + 确定性种子，本质就是在保证这种可回溯性。

C. 验证靠"分支对比"而非肉眼

switch + merge 高频，TA 常并排放"算法 A vs 算法 B"结果做对比。启发：Meshova 的 headless 截图闭环正是自动化版的"分支对比"——同种子跑两版脚本，截图比 IoU/差异，让 AI 客观判断哪版更好，而不是凭描述。

D. matchsize / bound：先量再变换

matchsize(68)、bound(21)、measure 频繁出现：专业流程在变换前先测量包围盒/尺寸，再做相对对齐，而非写死坐标。启发：Meshova 应提供 bbox()/centerOn()/fitInto() 工具，让 AI 脚本拼装多部件时用相对关系定位，鲁棒性远高于硬编码数值。
