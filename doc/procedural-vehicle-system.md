# Meshova 程序化载具系统

参考 Arjuna Ravikumar 的 Houdini 程序化载具思路：少量高层参数控制车身比例，分段轮廓构造主车体，再派生车窗、车轮、灯具、格栅和车型专属模块。实现为 Meshova 原生 TypeScript，不复制 Houdini 网络或资产。

## 生成链

`车型预设 → 纵向车身分段 → 横截面放样 → 车窗面组切线 → 独立内缩与向内挤出 → 轴距与轮组 → 前后细节 → 车型模块`

车窗不再是贴到车壳外的独立平面。生成器先从车身截面拓扑中移除对应窗面，再用同一组边界点生成窗框；玻璃按每个窗面独立内缩并向车内挤出。A/B/C 柱、窗框、车顶和腰线因此保持连续，参数变化时不会与玻璃脱离。

五种车型共用同一生成器：`sedan`、`suv`、`pickup`、`van`、`bus`。皮卡追加开放货箱；SUV/厢式车追加车顶行李架；巴士按车长分布七组侧窗。

## 核心参数

| 参数 | 作用 |
| --- | --- |
| `style` | 车身拓扑与比例规则 |
| `length` / `width` / `height` | 整车包围盒比例 |
| `wheelBase` | 前后轴距离 |
| `wheelRadius` / `wheelWidth` | 轮胎尺寸 |
| `rideHeight` | 离地与轮心高度 |
| `cabinPosition` | 驾驶舱前后偏移 |
| `roofRoundness` | 车顶横截面收束 |
| `hoodSlope` | 发动机盖到车顶过渡长度 |
| `detail` | 后视镜、门把手、行李架开关 |
| `seed` | 轮辐数量等离散细节，确定性复现 |

所有可见部件使用中文语义标签；英文部件名仅作稳定内部键。API 返回 `NamedPart[]`，可直接导出 OBJ/MTL、Viewer JSON，或进入截图/VLM 优化闭环。
