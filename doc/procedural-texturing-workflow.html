为什么要分流水线（不是一套打天下）

不同材质的“结构来源”根本不同，硬套一套流程会做出假东西。

  • 硬表面做旧（金属/石头/水泥）：结构来自 height，做旧靠 curvature/AO。→ height-first。
  • 有机纤维（布料/毛发/皮草）：结构来自方向场 + 各向异性，几乎没有“凸边磨损”。→ 另一套。
  • 分块拼接（砖/瓷砖/木地板）：核心是 tile 排布 + 每块随机化 + 缝隙，height 是次要。→ 又一套。
  • 分层涂装（做旧油漆/锈穿/掉漆）：核心是 mask 驱动的多层叠加，按暴露程度逐层剥。→ 再一套。
所以流水线是“可选策略”，不是唯一真理。本页把每种流水线命名、写清适用/禁区，AI 先选对策略，再调参，成品质量远高于无脑套一套。

AI 选择协议（生成贴图前先选流水线）

把“选流水线”作为材质生成的第一步，写进 agent 流程与 prompt。

  1. 分类：VLM / 文本意图 → 材质大类（硬表面/纤维/分块/涂装/透射）。
  2. 查表选流水线：用决策表拿到 pipeline id；命不中则回退到最接近的 + 标低置信度。
  3. 读该流水线的适用/禁区：禁区命中就换一条，别硬套。
  4. 按该流水线的步骤与参数空间生成：只在它暴露的参数里调，黑盒优化器也在这个空间搜。
  5. 评分回灌：类别匹配 + 粗糙/金属直方图 + 主色，分维度打分（造型优先、材质求类别对）。
落地建议：在 src/agent/api.ts 暴露 pickPipeline(materialClass) 与每条流水线的高层构建函数（如 buildPbrFromHeight），并把决策表 + 各流水线禁区写进 SCRIPT_API_REFERENCE，AI 才会先选后调而非乱拼裸函数。

如何新增一条流水线

学完一个新 SD 教程后，提炼成一条数据，不用碰 HTML 结构。

在本文件 PIPELINES 数组里加一个对象：

  • id：如 P2；name：流水线名；tag：一句话定位。
  • summary：它解决什么、核心思路。
  • steps：[{i,name,fn}] 有序步骤 + 对应 Meshova 函数。
  • fit / unfit：适用范围 与 禁区（最关键，AI 靠这个选）。
  • classes：命中的材质关键词（用于决策表）。

同步在“流水线选择决策表”补一行。保持深色风格不变。

所有流水线共用的心智模型

无论哪条流水线，底层都是这四类积木 + 三条铁律。

积木类别作用Meshova
发生器产生底图：噪声/图案/渐变/形状patterns / patterns2 / patterns3
滤镜改造单图：模糊/对比/扭曲/坡度/转法线filters · pbr
合成两图按遮罩混合blendTex / minTex / maxTex / blendColor
转换平铺/通道打包tiling · makeTile / tileSampler
三条铁律（跨流水线通用）：① 确定性——禁 Math.random/Date.now，走 makeRng().fork()。② 无缝——周期实现或过 makeTile。③ 物理——出口过 validateMaterial，材质只求类别对、不做像素复刻。
