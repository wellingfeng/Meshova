00总览：SpeedTree 的本质

SpeedTree 不是雕模型，而是一套参数化 + 程序化的递归生成器。一棵树 = 一组"生成器(Generator)"按父子层级嵌套，每层用样条(spline)描述骨架、用属性曲线(property curve)沿骨架调制半径/分叉/朝向，再把下一层"种"到上一层的样条上。这套思路和 Meshova 的 curve → sweep → copyToPoints 几乎一一对应。
Trunk 主干样条+半径曲线
→
Branch 枝沿父样条分布
→
Sub-branch 次枝递归
→
Leaf 叶/卡片末端散布
→
Wind 风顶点动画权重

核心可迁移的认知有四点：

  • 一切是样条：主干、枝、叶柄都是带半径的样条曲线，最终 sweep 成管状网格。
  • 层级递归：每个生成器把子生成器"实例化"到自己的样条上，参数随父代位置变化（越往末端越细、越密）。
  • 属性曲线驱动：几乎所有参数（半径、分叉角、长度、卷曲）都不是常数，而是一条 0→1 沿样条的曲线（Meshova 的 Ramp 正好是这个）。
  • 种子化随机：同一 seed → 同一棵树。这正是 Meshova 的硬约束，天然契合。
结论先行：树/灌木的骨架生成 80% 可用现有 API 拼出（curve/helix/sweep/copyToPoints/poissonScatter/ramp）。真正缺的是「递归分支生成器」「叶片卡片(cross-quad/billboard)」「风权重顶点属性」这三块，建议作为 P7「植被」模块新增。

02生成器分层：一棵树怎么"长"出来

SpeedTree 的生成器是一棵树形依赖图。理解这张图，就理解了它整个建模范式——递归地"把子件种到父件的样条上"。
Trunk 主干生成器 第0层
一条从根到顶的主样条。半径沿 t 从粗到细（property curve）。加 gnarl 噪声让它不直。Flare（根部外扩）单独一条曲线。输出 = 一根带锥度的扫掠管。
Branch 一级枝 第1层
在主干样条 Start%~End% 区间按 Frequency 采样 N 个生长点；每点建一条子样条，出射角 = Angle ± 随机，绕主干 Roll 螺旋分布（黄金角避免重叠）。长度/半径随父代 t 衰减。
Sub-branch 二/三级枝 递归
对每条一级枝重复同样规则。这就是递归：同一套"沿父样条散布子样条"逻辑，depth+1，参数缩放。通常 2~4 层后停止。
Leaf 叶片生成器 末端
在最末级枝的末段散布叶卡片/簇。每片是一个 quad（或交叉双 quad），朝向受法线+向上+随机抖动控制。远景用单张 billboard 代替全部叶子。
Frond 蕨叶/针叶 变体
棕榈/蕨/针叶树用"沿枝条连续生长的叶带"，本质是沿样条 sweep 一个扁平 profile，再贴 alpha 叶纹理。
Cap / Root / Knot 细节
封顶、板根、节疤等局部特征，多为额外散布的小网格或局部 displace。属于锦上添花，初版可跳过。

灌木与草：同一套生成器的退化形态

类型生成器配置关键差异
乔木(Tree)Trunk + 2~4 层 Branch + Leaf有明显主干，半径大，层级深
灌木(Shrub)多根细 Trunk(丛生) + 1~2 层 Branch + 密 Leaf无单主干，根部多杆丛生，整体偏球状
草(Grass)无 Branch，仅"叶片样条"+弯曲每根草 = 一条向上弯的窄样条扫掠/卡片，大量 poisson 散布成草地
藤蔓(Vine)Trunk 沿附着面爬行 + 稀疏 Leaf样条贴附表面，重力/吸附约束
关键洞察：草不是"小树"，而是"只有叶层的退化生成器"。所以一套统一的「样条+散布+卡片」生成器，通过参数即可覆盖树/灌木/草，无需三套代码。

03关键算法（可直接复现的部分）

这些是让植被"长得像真的"的核心数学，全部种子化、无外部依赖，符合 Meshova 决定论约束。

① 沿父样条分布子枝（黄金角螺旋）

子枝绕父样条的 roll 角用黄金角 137.5°递增，避免子枝在同一侧堆叠——这是植物叶序(phyllotaxis)的真实规律。沿父样条的位置用 start%~end% 区间均匀或抖动采样。

// 第 i 根子枝在父样条上的归一化位置 t 和绕轴角 roll
const t    = lerp(startPct, endPct, (i + jitter) / count);
const roll = i * 2.399963;  // 黄金角 (弧度) ≈ 137.5°
// 在父样条 t 处取位置 + 切线，建子枝起点坐标系
const P = sampleCurve(parent, t);
const T = curveTangent(parent, t);
const frame = frameFromTangent(T, roll);   // 切线+roll → 出射坐标系

② 出射方向：角度 + 向光 + 重力

子枝初始方向 = 父切线绕侧轴旋转 branchAngle；随后沿样条逐段把方向向上(向光性)或向下(重力)插值，得到自然的弯曲。

let dir = rotateAround(side, parentTangent, branchAngle);
for (let s = 0; s < segments; s++) {
  const up = vec3(0, 1, 0);
  dir = normalize(lerp(dir, up,      phototropism * dt)); // 向光上扬
  dir = normalize(lerp(dir, NEG_UP,  gravity      * dt)); // 重力下垂
  pt  = add(pt, scale(dir, segLen));
  pt  = add(pt, gnarlNoise(pt, seed));                    // 扭曲抖动
  pts.push(pt);
}

③ 半径锥度（property curve）

半径沿样条 t 不是线性，而是一条曲线：根部 flare 外扩、整体向梢部收细、末端尖。用 Ramp 表达最自然。

sweep(branchCurve, {
  sides: 6,
  radiusAt: (t) => baseRadius * radiusRamp.at(t)   // ramp: 0→1 的锥度曲线
});

④ 叶片朝向（交叉卡片 / billboard）

单叶 = 一个带 alpha 的 quad；为避免从侧面看变成一条线，常用交叉双 quad（cross）或 3 片 60° 互插。朝向由"枝条法线 + 世界向上 + 随机抖动"混合，让叶面大致朝外朝上。

const normal = mix(branchNormal, UP, leafUpBias);
const yaw    = rng() * TWO_PI;            // 绕自身轴随机转
const quad   = leafQuad(size, normal, yaw); // 朝向 normal 的卡片

⑤ 草叶弯曲（单条贝塞尔）

一根草 = 一条 3 点贝塞尔：根部直立，向风/重力方向弯，梢部最弯。窄 profile 扫掠或直接双面 quad 条。大量 poissonScatter 铺成草地，用密度图/噪声控分布。

const blade = bezier(
  root,                              // p0 根
  add(root, vec3(0, h*0.6, 0)),    // p1 控制点
  add(root, vec3(bend*0.5, h*0.9, 0)),
  add(root, vec3(bend, h, 0)), 6);    // p3 弯曲的梢

05风 / LOD / 卡片：渲染侧的工程

这部分是 SpeedTree 商业价值最高、也最"运行时"的部分。Meshova 现阶段（静态截图闭环）可先做"权重数据"，动画留给查看器。

风（Wind）

SpeedTree 的风是顶点着色器里的程序化摆动，不改网格拓扑。每个顶点带一个 wind weight ∈ [0,1]（根=0、梢=1，沿样条 t 累积），shader 用 sin(time + phase) * weight * windStrength 偏移顶点。分三层：主干整体摆、枝条次级摆、叶片高频抖。

Meshova 落地：生成网格时把 t（沿样条归一化深度）和 branchPhase 写进顶点属性，导出到 ViewerModel。three.js 端写一个简单 wind shader 读这些属性即可。静态截图不需要，但数据要预留。

LOD 与卡片(Billboard)

距离表现Meshova 做法
近(LOD0)全枝条 sweep + 每片叶 quadsweep sides=6~8，叶卡片全量
中(LOD1)枝降面，叶簇合并成少量大卡片sweep sides=3~4，叶片聚成 cluster quad
远(LOD2)整树→1~2 张交叉广告牌预渲染树的 8 方向图集 → cross billboard
广告牌(billboard)需要"把树渲染成贴图图集"，这正好能复用 Meshova 已有的 headless 多视角截图管线——把生成的树从 8 个角度截图拼图集，就是远景 imposter。是个很自然的二期增量。

06落地路线：P7「植被」模块建议

按 Meshova 一贯的"先骨架后细节、能截图即闭环"节奏，分四步增量交付。每步都能产出可在查看器看到的东西。
阶段交付物新增 API依赖
P7.1 样条骨架能长出一根带锥度、带噪声弯曲的主干curveFrameAt(curve,t)、gnarlCurve(curve,{seed,amount})curve/sweep/noise（全有）
P7.2 递归分支主干→多级枝的完整骨架树growBranches(parentCurve,{count,angle,depth,radiusRamp,...})P7.1 + 黄金角分布
P7.3 叶片末端散布叶卡片 → 看起来是棵真树leafCard(size,normal)、crossQuad()、scatterLeaves(branches,opts)P7.2 + poissonScatter
P7.4 灌木/草 + 风数据参数预设覆盖树/灌木/草；顶点写 wind weight预设 treePreset/shrubPreset/grassPreset；顶点属性通道P7.3 + 顶点属性扩展
建议从 P7.2 的 growBranches 切入做 MVP——它是整个范式的核心。先用现有 sweep 拼一棵"光杆树"，能在查看器里转着看，再加叶子。其余（板根、节疤、frond、风动画）都属可选增量。

与现有架构的接入点

  • 新建 src/vegetation/ 模块，barrel index.ts，由 src/index.ts 再导出（沿用现有约定）。
  • 把面向 AI 的函数注册进 src/agent/api.ts 的 SCRIPT_API 和 SCRIPT_API_REFERENCE（保持两者同步，这是硬约定）。
  • 叶片/树皮材质走现成 surfacePart + texture 模块，无需新东西。
  • 查看器加 tree ProcModel（schema 参数：seed/高度/分叉数/叶密度），滑块实时重建——延续现有程序化编辑器模式。
  • 决定论照旧：所有随机走 makeRng(seed).fork() 分流给各层，禁 Math.random。

07目标 DSL：一棵树应该怎么"写"出来

这是 P7 完成后，AI 脚本生成一棵树的理想样子——纯组合调用，无 import、无 async，符合 Meshova 的 DSL 契约。下面是设计草图（部分函数尚待实现），展示 API 该长什么样。
// 一棵橡树 —— P7 目标 DSL（草图，非现可运行）
const rng = makeRng(seed);

// 1) 主干：直立样条 + gnarl 扭曲 + 根部 flare
const trunk = gnarlCurve(
  polyline([vec3(0,0,0), vec3(0,4,0)]),
  { seed: rng.int(), amount: 0.15 }
);
const trunkMesh = sweep(trunk, {
  sides: 8,
  radiusAt: (t) => 0.3 * (1 - 0.85*t) + (t<0.1 ? 0.1*(1-t*10) : 0) // 锥度+flare
});

// 2) 递归分支：沿主干黄金角螺旋散布，3 级递归
const branches = growBranches(trunk, {
  seed: rng.int(),
  count: 7, depth: 3,
  startPct: 0.35, endPct: 0.95,
  angle: 50,            // 出射角(度)
  phototropism: 0.4,   // 向上弯强度
  lengthRamp: ramp([[0,1],[1,0.3]]),  // 越高的枝越短
  radiusScale: 0.6     // 每级半径衰减
});

// 3) 叶片：在末级枝散布交叉卡片
const leaves = scatterLeaves(branches.terminal(), {
  seed: rng.int(),
  density: 120, size: 0.18,
  upBias: 0.5, cross: true
});

// 4) 输出：树干/枝用树皮材质，叶用半透薄叶材质
return [
  surfacePart("trunk", merge(trunkMesh, branches.mesh()), "bark", { roughness: 0.9 }),
  surfacePart("leaves", leaves, "leaf", { transmission: 0.3, doubleSided: true })
];
对比可见：现在已能跑的部分是 polyline/sweep/merge/surfacePart；要补的是 gnarlCurve / growBranches / scatterLeaves / leafCard。把这几个补上，AI 就能像写上面这段一样生成无限变体的树。
