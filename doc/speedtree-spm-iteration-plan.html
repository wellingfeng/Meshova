0★ 流程诊断：给了 SPM 为什么生成还是这么差

核心问题不是“某个模块参数不好调”，而是现有闭环根本没读 SPM 里的任何数据。你手上有 1029 个 SPM 老师，但当前流程只把 SPM 当成了一个文件名，几何、骨架、叶位、颜色全部丢弃。这是天花板被锁死的真正原因。

✅ 已实现（本轮）：SPM = gzip 压缩的 SpeedTree XML，Node 内置 zlib 直接解，无需 SDK / Modeler / 逆向。
实测：文件头 1f 8b 08 即 gzip 魔数，解压后是 <SpeedTree Version="5"> XML，含 392 个命名参数 + 分层 Generator（Tree→Trunk→分支各级→Leaves）+ 数千个 Node 实例（每根枝干的样条骨架）。授权变体（基名 / _High / _Med / _Low / _RT）解析成功率 ≈ 98%；Desktop/Mobile 运行时变体是非标 gzip，自动跳过。
新增 scripts/spm-features.mjs 抽结构特征、src/vegetation/spm-fit.ts 映射为 Meshova 参数近真起点，speedtree-fit.mjs 接入 SPM 种子 + 参考图冠/干色分区采样（color 权重升到 0.28），30 棵树批量拟合 + 网页对比。这就是把下文“结构 / 颜色”两条断链接回闭环的落地实现。
实锤（来自 scripts/speedtree-fit.mjs 现状）：脚本里唯一涉及 .spm 的代码是 representativeSpm()——它做的只是在树种目录里挑一个 .spm 文件、取它的 basename 当 variant 字符串，然后拿这个名字去查一张手写的树种参数表。SPM 文件本身一个字节的几何数据都没有被读取。真正喂给评分的“参考”，是同目录里的一张预览 JPG 的剪影。

当前是“假闭环”：拿一张 2D 剪影，盲调一组和真树结构无关的手写参数

SPM只取文件名 → 树种名，几何全丢
手写表名字查一组默认参数，与真树结构无关
Meshova 生成从这个盲起点出发
预览 JPG只算剪影 IoU + 微弱颜色(0.12)
盲调 8 参坐标下降撑轮廓面积，不管形态

三条本该存在的信息通路，现在全断

信息通路SPM / 参考图里其实有现状后果
结构 主干曲线、一级分支角度分布、层级深度、冠幅/树高比、枝下高 全丢，只用文件名 优化起点靠瞎猜；单张 2D 剪影信息量喂不动 8 个参数
颜色 参考图里真实的冠色、干色 从不采样 叶色永远是硬编码随机绿；评分里颜色权重还只有 0.12
视角 可从多角度渲真值做对比 单张预览图 评分不稳；一个光滑实体球也能骗到高 IoU
叠加的造型硬伤：library.ts 的 broadleafCrownCloud() 用 实体 icosphere 球当叶冠主体——硬圆边、有高光、不通透，本质就是个“绿球”。而评分只奖励剪影面积吻合，绿球恰好轮廓 IoU 很高。于是优化器越调越像绿球，而不是越像真树。这就是截图里每棵阔叶树都糊成一坨的直接原因。

流程该怎么改：让 SPM 从“文件名”变成“真值目标”

不逆向、不复刻、不把 SPM 当运行时依赖。做法是离线一次性从 SPM 抽结构化统计特征当拟合真值——看老师的解题过程，不抄答案，输出仍然是纯 Meshova 脚本。这是把断掉的“结构 / 颜色”两条通路接回闭环。

离线导出v8.3 Modeler CLI 把 .spm 导成 Raw XML + OBJ
抽真值特征解析 XML → TreeFeature：骨架、角度直方图、冠干比；参考图分区采样冠/干色
映射初值TreeFeature → TreeRecipe 近真起点，不再瞎猜
多目标评分IoU + 破碎度 + 冠干比 + 色相，对齐真值
收敛入库坐标下降精修 → 树种参数库 → 生成新树

01 结论：v8.3 本地可做，先别等 SDK

SpeedTree Pipeline SDK 能读写原生 SPM，但不是你当前本机 v8.3 安装自带的必然能力。最稳开发路径：用 v8.3 Modeler 打开 SPM 并批量导出几何/Raw XML/贴图引用，再用 Meshova 自己的渲染与评分闭环逼近。

短期强路径：Modeler 导出

SpeedTree Modeler 命令行支持打开模型并导出，官方文档列出 -export、-export_game、-export_options。Windows 下用 Start-Process -Wait 等导出完成。

适配本机 v8.3可自动化

中期增强：Raw XML / custom exporter

Raw XML 可包含 Points、Vertices、Triangles、Leaf References、Bones、Meshes 等结构。它比 OBJ 更适合反推分枝骨架、叶片实例、材质 slot。

结构更完整需实测 v8.3 导出 preset

长期增强：Pipeline SDK

若拿到 Pipeline SDK，再读 SPM authoring graph。它能提高解析质量，但不能成为开源 Meshova 的运行时依赖，也不应该复制 SpeedTree 生成器。

可选增强授权边界
法务边界：不逆向破解 SPM 二进制；不复制贴图/网格到 Meshova 发布物；不把 SpeedTree generator 公式搬进 Meshova。只做本地离线参考、统计、截图评分、参数拟合。

05 特征抽取：从真实参考反推生成参数

特征抽取分三层：结构特征优先，截图特征兜底，VLM 只做类别与失败描述。不能让 VLM 单独决定参数。

XML/OBJ 结构层

  • Raw XML 若有 Bones / Leaf References：直接统计枝层、叶实例、材质 slot。
  • OBJ fallback：按 material/group 分 wood vs foliage；计算 AABB、冠层体素、连通组件。
  • 贴图文件名分类：Bark、Leaf、Normal、Spec、Branch、Thorn。
  • info.txt 提取 common name、latin name、生态描述。

截图评分层

  • 固定相机：front、side、top、orbit 45/135/225/315。
  • Pass：RGB、mask、depth、normal、matcap、wire、wood-only、foliage-only。
  • 指标：silhouette IoU、冠层密度、深度分布、轮廓曲率、孔洞率。
  • VLM：只输出“像什么树、哪里不像、优先调哪个语义参数”。

关键算法

特征抽取方法映射到 Meshova 参数
树高/冠宽/冠深 wood+foliage bounds；剔除离群叶卡；统一 Y-up。 height、crownScale、crownDepth
冠底高度 foliage 顶点按高度直方图，找连续高密区域起点。 canopy.baseY、branchStartPct
冠层形状 多高度切片半径曲线；拟合 cone/oval/umbrella/column/layered。 canopy.shape、branchLengthProfile
分枝层级 XML bones 优先；OBJ fallback 用 wood mesh skeletonize + 半径估计。 authoring.levels[]、branchCountProfile
枝条角度/下垂 branch tangent vs up 向量分布；按一级/二级/末端分桶。 angle、gravity、phototropism
叶团密度 leaf vertices 或 leaf refs 转 3D voxel；计算终端偏置、空洞率。 leafDensity、leafDensityProfile、terminalBias
材质族 贴图名 + 缩略图/VLM + 颜色直方图 + normal 频谱。 barkMaterial、leafMaterial、季节变体

当前生成器最该补的能力

能力为什么影响效果实现建议
Whorl / 层状分枝 针叶树不像，通常是枝层节奏错。 在 BranchPlacementMode 补 whorl phase、perWhorl jitter、层间距曲线。
冠层密度场 当前叶片容易像均匀噪声球。 按 crown envelope voxel 采样，控制空洞、外边缘稀疏、枝端聚集。
材质微结构 树皮/叶片质感不对，会让形体再好也像玩具。 树皮纵向 ridge/noise，叶片透光近似、背面色、roughness 范围。
年龄/季节变体 树库有 Young/Bare/Fall/RT/High/Low，能给模型学“同物种变体”。 把后缀转成 age、season、lodIntent 标签。
FitTarget 调参面板 AI 需要看差异、改语义参数，不应改 raw object 名。 Tree Lab 只显示“冠宽、冠底、枝角、叶密、下垂”等中文/语义 label。

07 评分与优化：多目标，不做像素复刻

最终目标是生成同类新树，所以评分要惩罚“死贴单棵参考”的过拟合。形态像、类别对、随机种子稳定，比像素级相同更重要。

分数权重实现调哪些参数
轮廓 IoU22%8 视角 mask；树干/冠层分区算。height、crownWidth、crownBase、lean
冠层密度18%体素密度 + 多视角 alpha 覆盖 + 空洞率。leafDensity、holes、terminalBias
枝架相似18%XML bones 或 wood skeleton；角度/层级/长度分布。branchLevels、angle、gravity
深度/法线统计12%depth histogram + normal orientation；不逐像素死比。crownDepth、branch curvature
材质族10%贴图名、颜色范围、normal 频率、VLM 粗判。barkProfile、leafProfile
类别语义10%VLM 判断：橡树/松树/棕榈/灌木是否稳定。classHint、placement、leaf type
生成稳定性10%同 recipe 换 16 seed，检查断枝、空冠、爆面、类别漂移。search space、constraints
 
score =
  0.22 * silhouette
+ 0.18 * crownDensity
+ 0.18 * branchSkeleton
+ 0.12 * depthNormal
+ 0.10 * materialFamily
+ 0.10 * vlmSpecies
+ 0.10 * seedStability
- penalties(overfit, intersections, brokenTopology, tooDense, tooFewParts)

08 MVP 样本：先做 12 棵，不吃完整树库

首批必须覆盖 4 类 grammar：圆冠阔叶、塔形针叶、棕榈/仙人掌、灌木/草。每类只锁 5-8 个核心参数，先让曲线正确。

类型本地样本拟合重点通过标准
圆冠阔叶 Broadleaves\Red_Oak、Broadleaves\American_Elm、Broadleaves\European_Beech 冠宽/冠底/枝角/叶团终端偏置。 8 视角 silhouette ≥ 0.70；VLM 稳定识别为阔叶树。
开伞/稀疏阔叶 Broadleaves\Acacia、Broadleaves\African_Baobab 伞形冠、粗干、枝端叶、空洞率。 不能生成成普通球冠；顶部轮廓要扁宽。
针叶树 Conifers\Colorado_Blue_Spruce、Conifers\Scots_Pine、Conifers\Italian_Cypress whorl、层状枝、锥形/柱形冠、针叶密度。 侧视锥形/柱形稳定；枝层可见。
棕榈/仙人掌 Palms_&_Cacti\Coconut_Palm、Palms_&_Cacti\Saguaro_Cactus 单干顶叶、frond fan、柱体分叉、刺/棱。 不能被识别成普通阔叶树。
灌木/草 Shrubs_&_Flowers\American_Boxwood、Shrubs_&_Flowers\Pampas_Grass 多主干、低矮冠、近地密度、blade clump。 换 seed 仍保持灌木/草类别。

09 开发路线

P0

树库 inventory + 现有 baseline 报告

复用 pnpm speedtree:library 生成 baseline，再新增 inventory。输出每类数量、变体、贴图族、样本缩略图、当前 Meshova 再生成截图。

1-2 天不碰 SPM 内部
P1

Modeler v8.3 导出验证

用 Acacia、Colorado Blue Spruce 各一棵验证 OBJ/XML 导出 preset。记录命令、输出结构、是否含 bones/leaf refs/material ids。

2-4 天决定 XML 价值
P2

Reference 包与渲染 pass

把导出 mesh 规范化到 Meshova/Three 渲染链，产出 RGB/mask/depth/normal/matcap。先不优化，只保证参考包可复现。

3-5 天截图回归基础
P3

特征抽取 + FitTarget

实现 bounds、冠层切片、叶团密度、分枝统计、材质族。输出 features.json 和可视化 HTML 报告。

1-2 周解释当前差距
P4

TreeRecipe 搜索

LHS + 局部爬山，先拟合 12 棵。每棵保存 top-10 recipes、分数曲线、失败原因。Tree Lab 支持人工锁定参数。

1-2 周闭环跑通
P5

参数先验库

按 broadleaf.round、broadleaf.umbrella、conifer.cone、conifer.column、palm.frond、shrub.clump 汇总 priors。新树从 priors 采样，不再拟合单棵。

持续生成新树

10 验收标准

阶段验收失败信号
P0 inventory 能稳定扫描 1029 个 SPM；baseline 能生成可浏览图库。 分类混乱、重复变体未归并、输出不可复现。
P1 至少 2 棵树导出 OBJ/XML 成功；导出日志可定位失败。 命令行导出不稳定、XML 无结构信息、尺度/朝向漂移。
P3 features 报告能解释样本：例如 Spruce 是锥形层状，Acacia 是伞形稀疏。 只得到 AABB，无法区分树种形态。
P4 12 棵中 8 棵达到类别正确 + silhouette ≥ 0.70 + 无明显拓扑坏点。 优化只会堆叶片；换 seed 类别漂移；针叶/阔叶互相串类。
P5 每个形态类能随机生成 20 棵新树，80% 通过 VLM 类别检查与基本几何检查。 只会复现训练样本，不会生成稳定新树。
